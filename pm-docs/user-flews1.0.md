
好的，我已经根据 `pm-docs/IA-design.md` 文件中的信息架构 (IA) 描述，为您绘制了以下几个核心用户流程图。这些流程图旨在可视化用户完成关键任务（如注册、登录、提交作品等）的步骤。

请注意，我已忽略 `pm-docs/user-flows.md` 文件。

以下是使用 Mermaid 语法绘制的用户流程图：

## 1. 用户注册流程 (User Registration Flow)

此流程描述了新用户如何在平台上创建账户。

```mermaid
graph TD
    A["访问平台"] --> B{"用户是否已注册?"};
    B -- "否" --> C["点击 '注册' 按钮"];
    C --> D["进入注册页面"];
    D --> E["输入手机号"];
    E --> F["点击 '获取验证码'"];
    F --> G{"验证码发送成功?"};
    G -- "是" --> H["输入短信验证码"];
    H --> I["同意用户协议"];
    I --> J["点击 '完成注册'"];
    J --> K{"注册成功?"};
    K -- "是" --> L["注册成功提示 & 自动登录"];
    K -- "否" --> M["错误提示"];
    M --> D;
    G -- "否" --> N["提示发送失败"];
    N --> E;
    B -- "是" --> O["跳转登录流程或已登录"];

    style L fill:#cfc,stroke:#333,stroke-width:2px
    style M fill:#f99,stroke:#333,stroke-width:2px
    style N fill:#f99,stroke:#333,stroke-width:2px
```

## 2. 用户登录流程 (User Login Flow)

此流程描述了已注册用户如何登录平台。

```mermaid
graph TD
    A["访问平台"] --> B{"用户是否已登录?"};

    D --> E["输入手机号"];
    E --> F["点击 '获取验证码'"];

    I --> J{"登录成功?"};

    J -- "否" --> L["错误提示"];
    L --> D;
    G -- "否" --> M["提示发送失败"];
    M --> E;
    B -- "是" --> N["已登录状态"];

    style K fill:#cfc,stroke:#333,stroke-width:2px
    style L fill:#f99,stroke:#333,stroke-width:2px
    style M fill:#f99,stroke:#333,stroke-width:2px
```

## 3. 作品提交流程 (Submission Flow)

此流程描述了用户登录后如何提交参赛作品。

```mermaid
graph TD
    subgraph "准备阶段"
        A["用户登录"] --> B["浏览赛事中心"];
        B --> C["选择具体赛道"];
        C --> D["查看赛道详情"];
    end

    subgraph "提交过程"
        D -- "点击 '立即参赛' 按钮" --> E;
        E["步骤1: 填写作品基础信息"] --> F["步骤2: 上传作品文件"];
        F --> G["步骤3: 填写团队成员信息 (可选)"];
        G --> H["步骤4: 预览与确认提交"];
        H --> I["点击 '确认提交'"];
    end

    subgraph "结果与后续"
        I --> J{"提交成功?"};
        J -- "是" --> K["提交成功提示 / 状态反馈"];
        K --> L["作品进入 '待审核' 状态"];
        L --> M["用户可在个人中心查看"];
        J -- "否" --> N["错误提示"];
        N --> E;
    end

    style K fill:#cfc,stroke:#333,stroke-width:2px
    style N fill:#f99,stroke:#333,stroke-width:2px
```

## 4. 浏览赛事与参赛流程 (Browse Competition & Participate Flow)

此流程描述了用户如何浏览不同的赛道并决定参赛。

```mermaid
graph TD
    A["访问首页"] --> B["点击 '赛事中心' 导航"];
    B --> C["进入赛道列表/概览页"];
    C --> D{"选择感兴趣的赛道?"};
    D -- "选择赛道" --> E["进入赛道详情页"];
    E --> F["了解赛道介绍、规则、时间轴"];
    F --> G{"决定参赛?"};
    G -- "是" --> H["点击 '立即参赛'"];
    H --> I{"用户是否已登录?"};
    I -- "是" --> J["跳转并开始作品提交流程"];
    I -- "否" --> K["提示登录/注册"];
    K --> L["执行用户登录流程"];
    L -- "登录成功" --> J;
    G -- "否" --> M["继续浏览其他赛道"];
    M --> C;
    E -- "返回赛道列表" --> C;
```

## 5. 浏览作品流程 (Browse Gallery/Showcase Flow)

此流程描述了用户如何浏览平台上的作品展示。

```mermaid
graph TD
    A["访问平台"] --> B["点击 '作品展示' 导航"];
    B --> C["进入作品展示墙主页"];
    C --> D{"使用筛选/排序功能?"};
    D -- "是" --> E["选择筛选条件 (赛道/年份/热度)"];
    E --> F["显示符合条件的作品列表"];
    D -- "否" --> F;
    F --> G{"点击某个作品?"};
    G -- "是" --> H["进入作品详情页"];
    H --> I["查看作品预览、介绍、作者信息"];
    I --> J{"分享作品?"};
    J -- "是" --> K["使用分享功能"];
    K --> H;
    J -- "否" --> L["返回作品列表"];
    L --> F;
    G -- "否" --> M["继续浏览或离开"];
    H -- "返回作品列表" --> F;
```

## 6. 个人中心管理作品流程 (Manage Submissions in User Center Flow)

此流程描述了登录用户如何在个人中心管理他们提交的作品。

```mermaid
graph TD
    A["用户已登录状态"] --> B["点击个人头像/用户名 或 导航至个人中心"];
    B --> C["进入个人中心"];
    C --> D["选择 '我的作品' 菜单"];
    D --> E["显示已提交作品列表"];
    subgraph "操作单个作品"
        E --> F{"选择操作"};
        F -- "查看详情" --> G["点击作品链接/查看按钮"];
        G --> H["显示作品详细信息与当前状态"];
        H --> I["返回作品列表"];
        I --> E;
        F -- "编辑作品 (截止日期前)" --> J["点击 '编辑' 按钮"];
        J --> K["进入作品编辑界面 (类提交流程)"];
        K --> L{"保存修改?"};
        L -- "是" --> M["更新作品信息"];
        M --> E;
        L -- "否" --> N["取消编辑"];
        N --> E;
        F -- "删除作品 (特定条件下)" --> O["点击 '删除' 按钮"];
        O --> P{"确认删除?"};
        P -- "是" --> Q["执行删除操作"];
        Q --> R{"删除成功?"};
        R -- "是" --> S["更新作品列表"];
        S --> E;
        R -- "否" --> T["显示错误信息"];
        T --> E;
        P -- "否" --> E;
    end
    E --> U["退出个人中心或浏览其他部分"];
```
